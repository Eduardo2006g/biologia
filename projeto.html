Introdução

O projeto Dog Scraper inicialmente chamado de Projeto Scraper tinha como estudo principal a coleta de informações e fotos utilizando técnicas como Web Scraping e posteriormente a validação dessas informações por meio de uma inteligência artificial treinada do zero a partir dos dados coletados na web. O projeto utilizou-se de fotos de variadas raças de cães devido a grande quantidade de informações disponíveis online, como APIs contendo desde fotos e características das raças e sub-raças apresentadas. No entanto, essas informações, por estarem dispersas por diferentes sites e páginas na web, a busca e validação manual desses dados seria no mínimo lenta, o que dificulta o processo de validação. O Web Scraping então surge como uma solução inicial para o problema, permitindo a busca de informações específicas relacionadas a cães pela web. Ao decorrer do projeto a Web Scraping aos poucos tornou-se pouco eficaz e o uso da Web Crawler passou a atender melhor os objetivos do projeto.

Objetivos

O Dog Scraper tem como objetivo: Desenvolver um sistema robusto que se beneficia de técnicas como web crawler para permitir a extração de informações diversificadas sobre cães a partir de APIs online confiáveis. Validar as informações coletadas, eliminando imagens ruins e outras inconsistências. Utilizar algoritmos de aprendizado de máquina para classificar, categorizar e validar as informações de acordo com critérios estabelecidos. Criar um modelo preditivo capaz de relacionar as informações coletadas com outras fotos de cães apresentadas, possibilitando a identificação de diferentes raças Porém o projeto em seu todo possui um objetivo mais ambicioso, que poderia vir a gerar um sistema capaz de coletar e validar informações e imagens sobre diferentes temas que poderiam ser tanto sobre outras espécies de animais quanto informações sobre notícias no geral

Materiais e Metódos

Com a utilização inicial de uma técnica chamada Web Scraper, a busca por imagens e dados relevantes para o projeto tornou-se muito eficaz, porém devido a alguns contrapontos o Web Crawler passou a ser a melhor opção na coleta de imagens. As imagens usadas no treino dos classificadores visuais foram coletadas de diversas fontes diferentes, a maioria delas APIs e sites contendo fotos de cães de diferentes raças e sub-raças. Entre os sites podemos citar a página “List of Dog Breeds” da Wikipédia https://en.wikipedia.org/wiki/List_of_dog_breeds) que por possuir um formato padrão de página, auxiliou o reconhecimento de outras páginas válidas e a transferência do algoritmo para extração de outros sites da Wikipédia, e a Dog API que fornece uma grande variedade de imagens de diversas raças.

Linguagens utilizadas:

JavaScript– Por conta de sua variedade de bibliotecas e sua eficácia na execução do código em comparação a outras linguagens, optou-se por sua utilização no desenvolvimento do Web Python– Algumas das redes neurais foram programadas em python e era usado no armazenamento de imagens no arquivo de saída do Scraper, além disso a linguagem foi muito utilizada pois permitia que arquivos fossem baixados e manipulados de forma simples no computador

Definição de Rótulos:

Ao utilizarmos os textos descritivos que acompanhavam as imagens coletadas, muitas vezes usados para expressar alguma característica da imagem apresentada, foi possível rotular imagens com descrições semelhantes e assim facilitar sua classificação

Treinamento de Classificadores:


Diferentes redes neurais feitas a partir de bibliotecas de código aberto como TensorFlow e Keras foram utilizadas nesse processo de classificação, permitindo assim, que houvesse uma confiabilidade muito maior quanto aos resultados considerando que diferentes redes neurais apresentaram resultados semelhantes.
